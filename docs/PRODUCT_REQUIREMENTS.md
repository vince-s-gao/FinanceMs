# InfFinanceMs 产品需求文档（PRD）

## 1. 文档信息

- 产品名称：InfFinanceMs
- 文档版本：v1.1
- 更新日期：2026-03-18
- 适用范围：Web 前端、API 后端、数据库与测试体系

## 2. 产品目标

InfFinanceMs 面向中小企业财务协同场景，围绕合同、回款、报销、费用与预算构建统一财务运营平台，提升业务透明度、执行效率与风险可控性。

## 3. 本次需求反写范围

本次反写覆盖以下已开发完成能力：

1. 数据导出功能
2. 消息通知
3. 自动化测试完善

## 4. 需求明细

### PRD-EXP-001 数据导出功能

**业务目标**

- 支持财务与管理角色快速导出报表数据，用于线下分析与归档。

**功能范围**

- 报表支持 CSV 导出：
  - 应收账款总览
  - 客户维度报表
  - 合同毛利分析
- 前端按当前报表 Tab 一键导出。
- 导出文件名包含日期，兼容中文文件名与 Excel 打开。

**验收标准**

- 用户在报表页点击导出按钮后，浏览器下载 CSV 文件。
- 后端返回 `Content-Type: text/csv; charset=utf-8`。
- 后端返回 `Content-Disposition` 且文件名可正确识别。
- CSV 内容字段与当前报表一致。

**实现映射**

- API：`GET /reports/export/receivables`、`GET /reports/export/customers`、`GET /reports/export/contracts/profit`
- 前端：报表页导出按钮与 Blob 下载逻辑
- 后端：CSV 格式化与 BOM 支持

### PRD-NOTI-001 消息通知

**业务目标**

- 为审批/付款等关键流程提供实时站内提醒，减少流程延迟与遗漏。

**功能范围**

- 通知列表查询（分页）
- 未读数查询
- 单条已读
- 全部已读
- 布局头部通知铃铛展示未读徽标与通知弹层
- 付款申请流程触发通知：
  - 提交后通知审批人
  - 审批后通知申请人
  - 确认付款后通知申请人

**验收标准**

- 登录后可查看个人通知列表与未读数。
- 点击通知可跳转目标页面并将消息标记已读。
- 可一键全部已读并即时刷新未读计数。
- 付款申请流转状态变化后，相关角色能收到对应通知。

**实现映射**

- API：`/notifications`、`/notifications/unread-count`、`/notifications/:id/read`、`/notifications/read-all`
- 数据模型：`Notification`、`NotificationType`
- 前端：主布局通知中心组件

### PRD-TEST-001 自动化测试完善

**业务目标**

- 建立稳定的质量门禁，降低回归风险。

**功能范围**

- 新增与补充以下模块测试：
  - `reports` 导出相关单测与控制器测试
  - `notifications` 服务层与控制器流程测试
  - `payment-requests` 通知联动分支测试
  - `auth` / `feishu` 边界分支覆盖
- 执行覆盖率门槛校验（含全局分支覆盖）。

**验收标准**

- 关键新增功能均有对应单元测试或 e2e-like 测试。
- `npm run test:cov --workspace=@inffinancems/api -- --runInBand` 通过。
- 覆盖率满足仓库阈值要求（全局分支覆盖不低于配置值）。
