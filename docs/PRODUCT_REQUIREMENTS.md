# InfFinanceMs 产品需求文档（PRD）

## 1. 文档信息

- 产品名称：InfFinanceMs
- 文档版本：v1.0.1
- 文档状态：当前实现反写（与仓库代码对齐）
- 更新日期：2026-03-18
- 覆盖范围：`apps/web`、`apps/api`、`packages/database`、`packages/shared`

## 2. 产品定位

InfFinanceMs 是面向中小企业的财务协同系统，以“合同为核心、现金流为主线、报销为费用入口”，统一管理客户、合同、回款、发票、报销、费用、预算与审批流程，提升经营透明度与财务执行效率。

## 3. 产品目标

1. 建立统一财务主数据与流程平台，减少跨系统切换。
2. 将合同-回款-开票-成本-毛利打通，形成经营可视化看板。
3. 通过审批流、权限控制、通知提醒与安全机制降低业务风险。
4. 通过高覆盖率自动化测试确保系统可持续迭代。

## 4. 目标用户与角色定义

系统内置角色：`EMPLOYEE`、`SALES`、`FINANCE`、`MANAGER`、`ADMIN`。

### 4.1 角色职责

- `EMPLOYEE`：报销申请与个人流程查看。
- `SALES`：客户与合同相关业务录入，项目协同。
- `FINANCE`：回款、发票、费用、预算、付款申请等财务执行。
- `MANAGER`：审批与经营数据查看。
- `ADMIN`：全局配置、组织权限、用户管理。

### 4.2 前端菜单可见性（当前实现）

- `EMPLOYEE`：`/dashboard`、`/expenses`
- `SALES`：`/dashboard`、`/customers`、`/contracts`、`/payments`、`/expenses`、`/projects`
- `FINANCE`：销售能力 + `payment-requests`、`invoices`、`costs`、`budgets`、`reports`
- `MANAGER`：与 `FINANCE` 基本一致
- `ADMIN`：全菜单 + `departments`、`permissions`、`settings`、`settings/dictionaries`、`audit-logs`

## 5. 业务域与功能全景

本版本覆盖如下业务域：

4. 合同与回款
5. 发票管理
6. 报销与费用
7. 预算管理
8. 付款申请

10. 消息通知
11. 附件上传
12. 审计与异常治理

## 6. 详细功能定义（按模块）

### FR-AUTH-001 认证与会话

**目标**

- 提供安全、可续期、支持企业 SSO 的登录体系。

**实现范围**

- 邮箱密码登录：`POST /auth/login`
- 飞书登录：授权 URL、回调、授权码登录、一次性 ticket 兑换
- 刷新令牌：`POST /auth/refresh`
- 当前用户：`GET /auth/me`
- 登出：`POST /auth/logout`
- CSRF token：`GET /auth/csrf`

**关键规则**

- `accessToken` 与 `refreshToken` 使用 Cookie 承载，`HttpOnly`。
- `csrfToken` 使用可读 Cookie，写操作需 `X-CSRF-Token` 头与 Cookie 一致。
- Access token 仅接受 `type=access`。
- Refresh token 仅接受 `type=refresh`。
- 用户被禁用时拒绝登录与令牌验证。

**验收标准**

- 未登录访问业务页自动跳转登录。
- token 过期可自动刷新，刷新失败则回登录页。
- 写操作缺失或错误 CSRF token 时返回禁止访问。

### FR-USER-001 用户管理

**范围**

- 用户分页查询、详情、创建、更新、禁用（软删除）
- 用户下拉选项（供部门配置等场景）

**规则**

- 邮箱唯一。
- 创建/更新密码均需 bcrypt 加密。
- 删除采用 `isActive=false`。

### FR-DEPT-001 部门管理

**范围**

- 部门 CRUD、树结构、启用禁用、成员管理、负责人设置

**规则**

- 同级部门名称不可重复。
- 不允许把自己或子部门设为父部门。
- 删除前必须无子部门。
- 设置负责人时若用户不在该部门，自动归属到部门。

### FR-PERM-001 权限管理

**范围**

- 菜单权限与功能权限定义
- 角色权限查询、更新、重置、初始化

**规则**

- 权限存储表：`role_permissions`
- 支持按角色批量重写菜单和功能权限
- 默认角色权限内置，支持覆盖

### FR-DICT-001 数据字典

**范围**

- 字典项 CRUD、按类型查询、类型列表、默认字典初始化
- 支持初始化合同类型（`CONTRACT_TYPE`）

**规则**

- `type + code` 唯一
- 同类型默认项唯一（更新默认时自动取消其他默认）

### FR-CUST-001 客户管理

**范围**

- 客户分页、详情、创建、更新、删除、审批、待审批列表、选项列表

**规则**

- 客户编号自动生成：`CUSxxxxxx`
- 信用代码唯一（软删除数据除外）
- 新建客户状态默认 `PENDING`
- 客户选项仅返回 `APPROVED`
- 已关联合同客户不可删除

### FR-PROJ-001 项目管理

**范围**

- 项目分页、详情、创建、更新、软删除

**规则**

- 项目编号自动生成：`TKFY{year}{seq}`
- 项目编号不可手工修改
- 删除采用 `isDeleted=true`

### FR-CONTRACT-001 合同管理

**范围**

- 合同分页、详情、创建、更新、状态流转、删除
- 合同主字段：
  - 合同编号
  - 签约年份（由签署日期自动推导）
  - 合同名称
  - 客户名称
  - 公司签约主体（`signingEntity`）
  - 合同类型（`contractType`，字典类型 `CONTRACT_TYPE`）
  - 合同金额
  - 签署日期
  - 结束日期

**规则**

- 合同编号自动生成：`HTYYYYMM-xxxx`
- 状态机：`DRAFT -> EXECUTING -> (COMPLETED|TERMINATED)`
- 仅 `DRAFT` 可编辑/删除
- 列表避免 N+1，回款聚合批量查询
- 列表查询支持以下筛选：
  - 合同名称/编号关键字
  - 客户名称模糊搜索
  - 签约年份
  - 合同类型
  - 签署日期范围（开始/结束）
- 新增/编辑合同支持录入“公司签约主体”和“合同类型”字段
- 合同列表支持按当前筛选条件导出 CSV
- 合同列表支持 CSV 批量导入（字段：合同名称、客户名称、公司签约主体、合同类型、合同金额、签署日期、结束日期）
- 批量导入采用“两段式”：预校验（不入库） -> 用户确认 -> 正式入库
- 导入支持常见中英文表头别名（如 `合同名称/name`、`客户名称/customer_name`、`签署日期/sign_date`）
- 页面提供“合同类型管理”入口，跳转数据字典并定位到 `CONTRACT_TYPE`

**验收标准**

- 合同列表可展示“签约年份、公司签约主体、合同类型、结束日期”字段。
- 按签约年份与合同类型筛选时，返回结果与条件严格匹配。
- 新建合同后，详情页可正确展示公司签约主体与合同类型。
- 导出文件字段包含：合同编号、签约年份、合同名称、客户名称、公司签约主体、合同类型、合同金额、签署日期、结束日期、状态。
- 批量导入返回导入摘要（总数、成功数、失败数）与失败行错误详情。
- 支持导入错误报告 CSV 下载（包含行号与错误原因），用于修复后二次导入。
- 预校验弹窗展示可导入样例（前 5 行）用于人工确认。
- 默认严格导入：存在错误行时禁止直接导入；用户可勾选“忽略错误并仅导入有效行”后继续。
- 提供导入历史（最近 10 次）查看与历史错误报告一键下载。
- 导入历史在刷新页面后仍可保留，且同账号在不同浏览器登录可查看一致结果。
- 导入历史持久化到后端数据库（`contract_import_logs`），支持跨浏览器/跨设备查看：
  - `ADMIN` 可查看与清理全量导入历史
  - `FINANCE` 仅可查看与清理本人导入历史

### FR-PAYMENT-001 回款管理

**范围**

- 回款统计、计划管理、记录管理、批量计划导入

**规则**

- 仅执行中合同可录入回款
- 回款累计不可超过合同应收
- 回款计划总额不可超过合同金额
- 回款记录增删会回算计划状态（`PENDING/PARTIAL/COMPLETED`）
- 回款变化后触发合同状态协调

### FR-INVOICE-001 发票管理

**范围**

- 发票分页、详情、开具、作废、合同开票风险预警

**规则**

- 发票号唯一
- 已开票累计不可超过合同金额
- 风险预警：`未开票金额 = 已回款 - 已开票`

### FR-EXPENSE-001 报销管理

**范围**

- 报销单分页、详情、创建、更新、提交、审批、打款、删除

**规则**

- 报销单号自动生成：`BXYYYYMM-xxxx`
- 员工仅可看/改/删自己的报销单
- 编辑态：`DRAFT` 或 `REJECTED`
- 提交态：需至少 1 条明细，状态转 `PENDING`
- 审批态：`PENDING -> APPROVED/REJECTED`
- 打款态：仅 `APPROVED` 可打款，状态转 `PAID`
- 打款后为每条明细生成费用记录（`source=REIMBURSEMENT`）
- 打款使用事务，防并发重复打款

### FR-COST-001 费用管理

**范围**

- 费用分页、详情、直接录入、删除、按合同汇总

**规则**

- 直接录入费用必须绑定项目，可选绑定合同
- 报销生成费用不可直接删除
- 查询支持按类型/来源/项目/合同/日期筛选

### FR-BUDGET-001 预算管理

**范围**

- 预算分页、详情、创建、更新、冻结/解冻、关闭、删除、部门汇总

**规则**

- 唯一约束：`year + month + department + feeType`
- 关闭预算不可编辑
- 有已使用金额预算不可删除
- 支持年度预算（`month=null`）与月度预算
- 提供预算使用金额更新能力（Service）

### FR-PAYREQ-001 付款申请

**范围**

- 付款申请创建、查询、统计、详情、更新、提交、审批、确认付款、取消、删除

**规则**

- 单号自动生成：`FKYYYYMMDDxxxx`
- 仅 `DRAFT` 可编辑/删除
- 提交流：`DRAFT -> PENDING`
- 审批流：`PENDING -> APPROVED/REJECTED`
- 付款流：`APPROVED -> PAID`
- 取消流：仅 `DRAFT/PENDING` 可取消

### FR-REPORT-001 报表看板

**范围**

- 应收账款总览
- 客户维度报表
- 报销分析
- 合同执行看板
- 合同毛利分析

**规则**

- 报表访问角色：`FINANCE/MANAGER/ADMIN`
- 工作台对 `EMPLOYEE` 和管理角色呈现不同视图

### FR-EXPORT-001 报表导出

**范围**

- 导出应收、客户、合同毛利 CSV
- 前端按当前 Tab 一键导出

**规则**

- `Content-Type: text/csv; charset=utf-8`
- 文件名按日期生成
- 附加 UTF-8 BOM 提升 Excel 兼容

### FR-NOTI-001 消息通知

**范围**

- 我的通知列表、未读数、单条已读、全部已读
- 顶栏通知中心（未读角标 + 列表 + 跳转）
- 付款申请流程事件通知

**规则**

- 通知类型：`SYSTEM/APPROVAL/PAYMENT/ALERT`
- 支持分页与仅未读过滤
- 前端轮询刷新（30 秒）

### FR-UPLOAD-001 附件上传

**范围**

- 支持合同/发票/报销/临时分类上传
- 提供删除与路径解析能力

**规则**

- 白名单 MIME + 扩展名 + 魔数三重校验
- 最大 10MB
- 防路径遍历与目录逃逸

### FR-AUDIT-001 审计与异常治理

**范围**

- 审计日志记录服务（含敏感字段脱敏）
- 日志管理页面（系统管理菜单下）
- 日志查询接口（分页、筛选、详情）
- 全局异常过滤器统一错误结构

**规则**

- 仅记录以下操作日志：`LOGIN`、`CREATE`、`UPDATE`、`DELETE`
- 不记录浏览日志（`GET/HEAD/OPTIONS` 请求不入库）
- 登录日志记录登录成功事件；增删改日志由全局拦截器自动采集
- 支持按操作类型、模块、操作人、关键字、时间范围筛选
- 统一错误返回：`{ code, message, details }`
- 生产环境隐藏内部异常细节

## 7. 核心业务流程定义

### 7.1 客户准入流程

1. 销售/财务创建客户，状态 `PENDING`
2. 管理层/管理员审批
3. 审批通过后客户进入可选列表并可用于合同创建

### 7.2 合同到回款流程

1. 财务创建合同（`DRAFT`）
2. 变更为执行中（`EXECUTING`）
3. 维护回款计划与回款记录
4. 系统自动回算应收、计划状态与合同执行状态

### 7.3 报销到费用流程

1. 员工创建报销草稿并提交
2. 财务/管理员审批
3. 审批通过后打款
4. 系统自动生成费用明细（来源 `REIMBURSEMENT`）

### 7.4 付款申请流程

1. 财务创建付款申请草稿
2. 提交后通知审批人（管理层/管理员）
3. 审批通过/驳回后通知申请人
4. 财务确认付款后通知申请人

### 7.5 报表与导出流程

1. 财务/管理层进入报表看板
2. 查看应收、客户、毛利、报销等分析
3. 按当前页签导出 CSV

## 8. 权限与访问控制

### 8.1 后端访问控制

- 默认接口由 `JwtAuthGuard` 保护
- `@Public` 仅用于登录、刷新、CSRF、飞书授权等必要公开接口
- `RolesGuard` 基于 `@Roles(...)` 做角色校验

### 8.2 前端路由控制

- 中间件基于 Cookie 中 JWT 有效期进行预校验
- 公共路由：`/login`、`/login/callback`
- 受保护路由无 token 时重定向登录并携带 `redirect`

## 9. 数据模型定义（核心）

核心实体：

- 用户与组织：`User`、`Department`、`RolePermission`
- 业务主数据：`Customer`、`Project`、`Dictionary`、`BankAccount`
- 合同主链路：`Contract`、`PaymentPlan`、`PaymentRecord`、`Invoice`
- 费用主链路：`Expense`、`ExpenseDetail`、`Cost`、`Budget`
- 付款与通知：`PaymentRequest`、`Notification`
- 审计：`AuditLog`

关键状态枚举：

- `ContractStatus`、`PaymentPlanStatus`、`InvoiceStatus`
- `ExpenseStatus`、`PaymentRequestStatus`
- `BudgetStatus`、`NotificationType`

## 10. 接口与前端交互规范

1. API 全局前缀：`/api`
2. 前端统一通过 axios 客户端调用，`withCredentials=true`
3. 401 时自动触发 refresh（排除登录/刷新/登出相关路径）
4. 并发请求刷新时采用队列等待，避免重复刷新
5. 统一错误映射为 `{ status, code, message, details }`

## 11. 安全与合规要求

1. Cookie 安全属性：`HttpOnly`、`SameSite`、生产环境 `Secure`
2. CSRF：双提交 Cookie + Header 校验
3. 安全头：Helmet（CSP、HSTS 等）
4. 速率限制：
   - 全局限流（生产默认 15 分钟 100 次）
   - 登录接口更严格限流（生产默认 15 分钟 5 次）
5. 上传安全：MIME/扩展名/魔数校验 + 路径防护
6. JWT 密钥强度校验：生产环境弱密钥禁止启动

## 12. 非功能性要求

### 12.1 可用性

- 前端关键页面支持加载态、空态、错误提示
- 通知与仪表盘支持周期刷新

### 12.2 性能

- 分页查询默认支持 10/20/50/100（后端 DTO 限制 `pageSize<=100`）
- 报表导出支持按需下载，不阻塞页面渲染

### 12.3 可维护性

- Monorepo 分层：`apps` + `packages`
- 共享常量、类型与错误码统一在 `@inffinancems/shared`

## 13. 质量与测试基线

1. API 侧包含 service 单测与 e2e-like 测试
2. 覆盖率门槛（Jest 配置）：
   - `branches >= 99%`
   - `functions >= 99%`
   - `lines >= 99%`
   - `statements >= 99%`
3. 关键新增能力（导出、通知、刷新与安全分支）已纳入自动化测试

## 14. 部署与运行要求

1. Node.js >= 18，PostgreSQL >= 14
2. 后端默认端口：`3001`，前端默认端口：`3000`
3. Swagger 文档地址：`/api/docs`
4. 数据库变更需同步 Prisma migration
